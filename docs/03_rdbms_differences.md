# RDBMS-eltérések

Ez a dokumentum az eddigi SQL-szkriptek alapján összefoglalja a legfontosabb eltéréseket az egyes adatbázis-kezelők között.

A cél nem teljes SQL referencia készítése, hanem a projektben ténylegesen használt példák alapján látható különbségek összegyűjtése.

## Automatikus azonosítók

Az automatikusan növekvő technikai kulcsok adatbázisonként eltérő szintaxissal szerepelnek.

| Adatbázis  | Példa                                      |
| ---------- | ------------------------------------------ |
| Oracle     | `NUMBER GENERATED BY DEFAULT AS IDENTITY`  |
| SQL Server | `INT IDENTITY(1,1)`                        |
| PostgreSQL | `INTEGER GENERATED BY DEFAULT AS IDENTITY` |
| MySQL      | `INT AUTO_INCREMENT`                       |
| IBM Db2    | `INTEGER GENERATED BY DEFAULT AS IDENTITY` |
| SQLite     | `INTEGER PRIMARY KEY AUTOINCREMENT`        |

## Aktuális dátum/idő lekérdezése

Az első kapcsolattesztekben minden adatbázisban egyszerű dátum/idő lekérdezés futott.

| Adatbázis  | Lekérdezés                                        |
| ---------- | ------------------------------------------------- |
| Oracle     | `SELECT SYSDATE FROM dual;`                       |

## Cleanup / táblák törlése

A táblák törlésekor fontos a sorrend, mert az `orders` tábla idegen kulccsal hivatkozik a `customers` táblára.

Ezért először az `orders`, utána a `customers` tábla törölhető.

```sql
DROP TABLE orders;
DROP TABLE customers;
```

Egyes adatbázisok támogatják az `IF EXISTS` kiegészítést is:

```sql
DROP TABLE IF EXISTS orders;
DROP TABLE IF EXISTS customers;
```

## Séma és adatbázis fogalom

A projektben az adatbázis / séma fogalma is eltérően jelenik meg.

- Oracle esetén a felhasználó / séma szerepe erősebb.
- SQL Server esetén külön adatbázis és azon belül sémák használhatók.
- PostgreSQL esetén adatbázison belül sémák vannak, alapértelmezésben gyakran a `public` séma.
- MySQL-ben a schema és database fogalma a gyakorlatban sokszor azonosként jelenik meg.
- IBM Db2 esetén a séma külön névtérként jelenik meg.
- SQLite esetén nincs klasszikus szerveroldali adatbázis/séma logika, a működés fájlalapú.
